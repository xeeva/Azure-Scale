How to Use This Graph

What You're Looking At

Each large circle is an Azure resource provider (e.g. Compute, Network). Small circles are sub-services and products within a provider. Lines show dependencies and relationships between services.

Navigation

  • Pan — click and drag the background
  • Zoom — scroll wheel or pinch
  • Inspect — click any node to open its detail panel
  • Move nodes — drag a node to reposition it
  • Reset View — re-centre and zoom to fit
  • Freeze — pause the physics simulation

Sidebar Filters

  • Search — find providers and sub-services by name
  • Categories — toggle Compute, Networking, Data, etc.
  • Sub-Services — show or hide sub-service nodes
  • Preview APIs — highlight providers that only have preview-stage APIs
  • Regions — filter nodes by Azure region availability

Legend

  • Node size = complexity score (config params + SKUs + ops)
  • Colours = category (Compute = blue, Networking = cyan, etc.)
  • Thick lines = provider-to-provider dependency
  • Thin lines = parent ↔ sub-service link
  • Dashed lines = cross-service relationship

Enhanced Features

  • Stats Bar — live count of visible providers, sub-services, relationships, and parameters
  • Heatmap — colour nodes by complexity, SKU count, or operations
  • Human Cost — overlay showing time-to-learn estimates per provider
  • Licensing — filter by license tier; see discount details in the node panel
  • Centrality — badges showing the most connected providers
  • Comparison — select two providers and compare side-by-side
